# Systematic Alpha Research Lab — Outputs and Reporting Specification

## Output Objective

Generate all research outputs from code and present them as clear evidence for each signal verdict.

The website must not hardcode results. Every metric, chart, note, and classification must come from generated research outputs.

## Output Pipeline

```text id="snh9rw"
Data
→ Signals
→ Backtests
→ Validation
→ Classification
→ Website JSON
→ README / Report
```

## Output Structure

```text id="ncc406"
research/
  outputs/
    data_audit/
    signals/
    backtests/
    validation/
    website_json/
    reports/
```

## Output Principles

All outputs must be:

* generated by code
* reproducible
* traceable to source data
* consistent across website and README
* concise
* verdict-oriented
* easy to inspect

Do not generate unnecessary files.

## Website Reporting Rules

The website must report:

* what was tested
* what happened net of costs
* whether it beat benchmarks
* whether it survived out-of-sample
* what broke under stress
* final verdict

The website must not display every raw output.

## Research Report

Generate a concise report:

```text id="k2lzdg"
research/outputs/reports/research_report.md
```

Required sections:

1. Research Question
2. Data and Universe
3. Signal Cases
4. Backtest Methodology
5. Validation Results
6. Final Classification
7. Limitations

Do not create a long academic paper.

## README Outputs

The README must include:

* project summary
* research question
* signal list
* data note
* validation summary
* final verdict table
* how to reproduce
* limitations
* not investment advice

## Hard Rules

* Do not hardcode metrics in the website.
* Do not report gross performance without net performance.
* Do not hide failed signals.
* Do not bury conclusions in appendices.
* Do not create excessive charts.
* Do not make the report longer than needed.

Outputs succeed if a viewer can trace each verdict back to generated evidence.
